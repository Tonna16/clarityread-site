Quick troubleshooting

  • Ensure the extension is enabled in chrome://extensions.
  • Try disabling other reader or accessibility extensions — they may conflict.
  • If read-aloud fails, check your browser's speech voices and permissions.

Debugging & diagnostics (privacy-first)

I’ve disabled automatic console logs by default to protect privacy. If you run into a problem, use these safe options to help me debug without sending page content.

  • Click copy diagnostics — the extension offers a debug toggle in the popup. When clicked it collects minimal diagnostics (extension version, browser user agent, recent extension errors) and allows to paste into an issue or email. It is not required but strongly recommended that you do this if you run into an issue.
  • Reproduction steps — list exactly: site URL, steps you took, what you expected, and what happened. This is often enough to fix many issues.
  • Screenshots — a screenshot of the problem and the extension UI is often the fastest way to debug.
